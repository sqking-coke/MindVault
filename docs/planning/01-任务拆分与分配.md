## Sub-Issue 创建计划

> 在 Q 确认规划后，按以下策略创建 sub-issues：

### Batch 1: 接口契约（立即创建，串联前后端）
- **T-00**: assignee = Multica, status = todo

### Batch 2: 前后端并行（T-00 完成后创建）
- **T-B01~T-B08**: assignee = Claude (新建专用 agent), status 按序设置为 todo/backlog
- **T-F01~T-F06**: assignee = Gemini, status 按序设置为 todo/backlog
- **T-I01~T-I04**: assignee = Multica, status = backlog

### Batch 3: 联调验收（前后端就绪后创建）
- **T-V01~T-V02**: assignee = Multica, status = todo

## 序列化任务 vs 并行

| 类型 | 策略 | 示例 |
|------|------|------|
| 严格串行 | T-B01→02→03→04→05→06 数据流有前后依赖，每次只启动一个 | 后端 RAG 链路 |
| 可并行 | T-B01 和 T-F01 无依赖，可同时启动 | 前后端并行 |
| 批次并行 | T-I02/I03/I04 都依赖 T-I01 但不互相依赖 | 基础设施三件套 |
