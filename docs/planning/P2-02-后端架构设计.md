## 4. 核心新增模块详细设计

### 4.1 Redis 缓存服务 (`cache_service.py`)

```python
# 设计要点
class CacheService:
    """Redis 检索结果缓存"""
    
    async def get_retrieval(redis, query_hash: str) -> list[RefChunk] | None:
        """按 query embedding hash 读取缓存"""
        ...
    
    async def set_retrieval(redis, query_hash: str, chunks: list[RefChunk], ttl: int = 3600):
        """写入缓存，TTL 默认 1 小时"""
        ...
    
    async def invalidate_document(redis, doc_id: int):
        """文档更新/重索引后，清除相关缓存（pattern: retrieval:*）"""
        ...
```

**缓存 Key 设计**:
- 格式: `retrieval:{md5(embedding_bytes)}`
- TTL: 3600 秒（环境变量 `REDIS_CACHE_TTL`）
- 序列化: JSON (通过 RefChunk.model_dump())

**集成点**: `retrieval_service.retrieve_chunks()` — 检索前先查缓存，命中则直接返回；miss 则查 pgvector 后写缓存

## 5. 数据流: P2 增量重索引

```mermaid
sequenceDiagram
    participant FE as 前端
    participant API as /documents/{id}/reindex
    participant DS as document_service
    participant Cache as Redis
    participant IS as ingestion_service
    participant PS as parser_service
    participant ES as embedding_service
    participant DB as PostgreSQL

    FE->>API: POST /api/v1/kb/documents/42/reindex
    API->>DS: reindex_document(db, 42)
    DS->>DB: SELECT document WHERE id=42
    DS->>DB: DELETE FROM kb_chunks WHERE document_id=42
    DS->>Cache: FLUSHALL retrieval:*
    DS->>DB: UPDATE document SET status=1 (processing)
    DS-->>API: {"doc_id": 42, "status": "processing"}
    API-->>FE: 202 Accepted

    Note over IS: 后台异步任务
    IS->>PS: parse_document (逐页)
    PS-->>IS: [(text, page), ...]
    IS->>IS: chunk_text (语义切片)
    IS->>ES: embed_text (逐条)
    ES-->>IS: [float x 1024]
    IS->>DB: INSERT chunks (含 page 字段)
    IS->>DB: UPDATE document SET status=2, chunk_count=N
```

## 6. 数据流: Redis 缓存命中

```mermaid
sequenceDiagram
    participant Chat as chat_service
    participant RS as retrieval_service
    participant Cache as Redis
    participant DB as PostgreSQL (pgvector)

    Chat->>RS: retrieve_chunks(db, embedding)
    RS->>RS: query_hash = md5(embedding_bytes)
    RS->>Cache: GET retrieval:{query_hash}
    
    alt 缓存命中
        Cache-->>RS: [RefChunk JSON]
        RS-->>Chat: 反序列化后的 chunks
    else 缓存未命中
        RS->>DB: SELECT ... cosine_distance ... HNSW
        DB-->>RS: [(chunk_id, doc_name, content, similarity), ...]
        RS->>Cache: SET retrieval:{query_hash} = JSON(chunks) EX 3600
        RS-->>Chat: chunks
    end
```
