## 1. 数据库设计

### 1.1 表结构 DDL

基于设计文档 v1.1 第 7 章，直接翻译为 Alembic migration。

```sql
-- 启用 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 知识库文档主表
CREATE TABLE kb_document (
    id          BIGSERIAL PRIMARY KEY,
    doc_name    VARCHAR(255) NOT NULL,
    doc_type    VARCHAR(50)  NOT NULL,  -- txt, md, pdf, docx
    doc_desc    TEXT,
    file_path   VARCHAR(500) NOT NULL,
    status      SMALLINT DEFAULT 1,     -- 0=禁用, 1=启用
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    updated_at  TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_kb_document_status ON kb_document(status);

-- 知识切片数据表
CREATE TABLE kb_chunk (
    id            BIGSERIAL PRIMARY KEY,
    doc_id        BIGINT NOT NULL REFERENCES kb_document(id) ON DELETE CASCADE,
    chunk_content TEXT NOT NULL,
    chunk_index   INT NOT NULL,
    embedding     vector(1024),          -- pgvector, 1024维对应 BGE-large-zh-v1.5
    status        SMALLINT DEFAULT 1,    -- 0=失效, 1=有效
    created_at    TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_kb_chunk_doc_id ON kb_chunk(doc_id);
CREATE INDEX idx_kb_chunk_embedding ON kb_chunk
  USING hnsw (embedding vector_cosine_ops)
  WITH (m = 16, ef_construction = 200);

-- 会话表
CREATE TABLE kb_session (
    id         BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL UNIQUE,  -- UUID
    title      VARCHAR(255) DEFAULT '新会话',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE UNIQUE INDEX idx_kb_session_sid ON kb_session(session_id);

-- 问答记录表
CREATE TABLE kb_qa_record (
    id         BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL REFERENCES kb_session(session_id),
    question   TEXT NOT NULL,
    answer     TEXT NOT NULL,
    ref_chunks JSONB DEFAULT '[]',
    model_name VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_kb_qa_record_session ON kb_qa_record(session_id);

-- 知识库配置表
CREATE TABLE kb_config (
    id           BIGSERIAL PRIMARY KEY,
    config_key   VARCHAR(100) NOT NULL UNIQUE,
    config_value TEXT NOT NULL,
    config_desc  VARCHAR(255),
    updated_at   TIMESTAMPTZ DEFAULT NOW()
);

-- 预置配置
INSERT INTO kb_config (config_key, config_value, config_desc) VALUES
  ('embedding_dim', '1024', 'Embedding 向量维度'),
  ('chunk_size', '512', '默认切片大小'),
  ('chunk_overlap', '64', '默认切片重叠长度'),
  ('top_k', '5', '检索召回数量'),
  ('similarity_threshold', '0.7', '相似度过滤阈值');
```

### 1.2 索引策略

| 表 | 索引 | 类型 | 用途 |
|----|------|------|------|
| kb_document | `status` | B-tree | 按状态过滤文档 |
| kb_chunk | `doc_id` | B-tree | 按文档查询切片 |
| kb_chunk | `embedding` | **HNSW** | 高性能向量相似度检索 |
| kb_session | `session_id` | UNIQUE B-tree | 按 UUID 查会话 |
| kb_qa_record | `session_id` | B-tree | 按会话查历史 |
| kb_config | `config_key` | UNIQUE B-tree | 按键查配置 |

### 1.3 Alembic 迁移策略

```
backend/
├── alembic/
│   ├── versions/
│   │   └── 001_init_schema.py    ← 首次迁移：创建所有表 + pgvector 扩展
│   └── env.py                     ← 配置 async engine
└── alembic.ini
```

启动时自动执行 `alembic upgrade head`，无需手动迁移。

### 1.4 pgvector 参数预设与调优

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `m` | 16 | HNSW 每层最大连接数，影响构建速度与召回精度 |
| `ef_construction` | 200 | 构建时搜索宽度，越大索引质量越高但构建越慢 |
| `ef_search` | 40 | 查询时搜索宽度，可通过 `SET hnsw.ef_search = 40` 动态调整 |
| `maintenance_work_mem` | 512MB | 索引构建内存分配 |

```sql
-- 会话级调优（在检索查询前执行）
SET hnsw.ef_search = 40;
```

## 2. 部署架构

### 2.1 Docker Compose 拓扑

```
┌─────────────────────────────────────────────────────┐
│                    nginx:80                          │
│  /          → frontend:3000                         │
│  /api/*     → backend:8000  (buffering off for SSE) │
└──────────┬────────────────────┬─────────────────────┘
           │                    │
    ┌──────▼──────┐      ┌──────▼──────┐
    │  frontend   │      │  backend    │
    │  next.js 14 │      │  FastAPI    │
    │  :3000      │──────│  :8000      │
    └─────────────┘      └──────┬──────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
        ┌─────▼─────┐   ┌──────▼──────┐   ┌──────▼──────┐
        │ PostgreSQL │   │   Redis     │   │   Ollama    │
        │ pgvector   │   │   :6379     │   │   :11434    │
        │ :5432      │   │   (缓存)    │   │   (LLM+Emb) │
        └────────────┘   └────────────┘   └─────────────┘
```

### 2.2 Docker Compose 配置

```yaml
# docker-compose.yml (项目根目录)
version: '3.8'

services:
  frontend:
    build:
      context: .
      dockerfile: Dockerfile.frontend
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_BASE=http://localhost/api/v1
    depends_on:
      - backend
    restart: unless-stopped

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    env_file:
      - .env
    environment:
      - DATABASE_URL=postgresql+asyncpg://mindvaults:mindvaults@db:5432/mindvaults
      - REDIS_URL=redis://redis:6379
      - LLM_BASE_URL=http://ollama:11434/v1
    volumes:
      - uploads_data:/app/uploads
      - ./logs:/app/logs
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
      ollama:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: pgvector/pgvector:pg16
    environment:
      POSTGRES_DB: mindvaults
      POSTGRES_USER: mindvaults
      POSTGRES_PASSWORD: mindvaults
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U mindvaults"]
      interval: 5s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    restart: unless-stopped

  ollama:
    image: ollama/ollama:latest
    volumes:
      - ollama_data:/root/.ollama
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:11434/api/tags"]
      interval: 10s
      timeout: 5s
      retries: 10
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - frontend
      - backend
    restart: unless-stopped

volumes:
  pgdata:
  ollama_data:
  uploads_data:
```

### 2.3 后端 Dockerfile

```dockerfile
# backend/Dockerfile
FROM python:3.11-slim

WORKDIR /app

# 系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential curl \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# 启动脚本：执行数据库迁移 + 启动服务
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

EXPOSE 8000

ENTRYPOINT ["/entrypoint.sh"]
```

### 2.4 启动入口脚本

```bash
#!/bin/bash
# backend/entrypoint.sh

set -e

echo "==> Waiting for PostgreSQL..."
until pg_isready -h db -U mindvaults; do sleep 1; done

echo "==> Running Alembic migrations..."
cd /app
alembic upgrade head

echo "==> Starting FastAPI..."
exec uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 2.5 nginx 配置

```nginx
# nginx.conf
events { worker_connections 1024; }

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    server {
        listen 80;

        # 前端
        location / {
            proxy_pass http://frontend:3000;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
        }

        # 后端 API（SSE 流式端点必须关闭缓冲）
        location /api/ {
            proxy_pass http://backend:8000;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_buffering off;
            proxy_cache off;
            proxy_set_header X-Accel-Buffering no;
            proxy_set_header Connection '';
            proxy_read_timeout 300s;
        }
    }
}
```

## 3. CI/CD 方案（可选 P0）

> P0 阶段可选，仅做快速验证。P1 配置正式 CI/CD。

### 3.1 GitHub Actions 工作流

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend-tests:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: pgvector/pgvector:pg16
        env:
          POSTGRES_DB: mindvaults_test
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: |
          cd backend
          pip install -r requirements.txt
          pip install pytest pytest-asyncio httpx
          alembic upgrade head
          pytest

  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run build
      - run: npm run lint
```

## 4. 首次部署流程

```bash
# 1. 克隆项目
git clone git@github.com:sqking-coke/mindvaults.git
cd mindvaults

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，设置 API_KEY、数据库密码等

# 3. 启动服务
docker compose up -d

# 4. 等待 Ollama 就绪 + 模型拉取
# Ollama 容器首次启动需下载 qwen3 + nomic-embed-text（替换为 BGE-large-zh-v1.5）
docker exec -it mindvaults-ollama-1 ollama pull qwen3
docker exec -it mindvaults-ollama-1 ollama pull nomic-embed-text

# 5. 验证
curl http://localhost/api/v1/health
# → { "code": 0, "data": { "status": "ok", "database": "connected", ... } }

# 6. 上传测试文档
curl -X POST http://localhost/api/v1/kb/documents \
  -F "files=@test.md"

# 7. 测试问答
curl -X POST http://localhost/api/v1/kb/chat \
  -H "Content-Type: application/json" \
  -d '{"question":"测试问题","session_id":"test-123"}'
```
