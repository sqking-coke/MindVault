# 00 - MindVault P0 任务规划总览

> 基于设计文档 v1.1，本文档定义 P0 MVP 阶段的总体策略、并行路线和里程碑节点。

## 执行工作流

```
任务规划 ──► 任务拆分 ──► 任务分配 ──► 细分文档完善 ──► 沟通确认 ──► 实施
 (本文档)     (01-文档)    (01-文档)   (02~05-文档)     (本评论)    (sub-issues)
```

## 当前状态矩阵

| 维度 | 状态 | 详情 |
|------|------|------|
| 前端代码 | Mock 原型就绪 | Next.js 14 App Router, Chat(400行)/KB(800行)/Context(600行), 需重构 |
| 后端代码 | **零** | 无 API、无 DB 连接、无 RAG 能力 |
| 数据库 | **零** | 无 PostgreSQL/pgvector 实例 |
| 基础设施 | **零** | 无 Docker Compose、无 CI/CD |
| 设计文档 | 完备 | v1.1 覆盖架构/接口/DB/部署, 1000+ 行 |
| 重构计划 | 完备 | REFACTOR_PLAN.md 定义 6 项前端任务 |

## P0 目标：RAG 最小闭环

> 跑通「**文档上传 → 解析切片 → 向量入库 → 单轮问答**」完整链路。

## 总工期估计

| 工作线 | 任务数 | 预估工期 | 负责人 |
|--------|--------|---------|--------|

## 里程碑

| 节点 | 交付物 | 验收标准 |
|------|--------|---------|
| M1: 规划确认 | 6 份规划文档 | Q 审阅通过 |
| M2: 契约锁定 | `src/types/api.ts` + OpenAPI 规范 | 前后端一致认可 |
| M3: 后端就绪 | FastAPI + pgvector + RAG 链路 | `POST /api/v1/kb/chat` 返回正确答案 |
| M4: 前端就绪 | 重构后 Chat/KB 页面 + API 对接 | 与后端联调通过 |
| M5: 部署就绪 | Docker Compose 一键启动 | 本地 `docker compose up` 完整可用 |
| **M6: MVP 上线** | **RAG 最小闭环可演示** | **上传文档 → 问答 → 引用溯源全链路通过** |

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Ollama 模型下载慢 | 高 | 低 | 提前配置国内镜像或离线加载 |
| 前后端接口对接偏差 | 中 | 中 | 接口契约先行，锁定后再并行开发 |
| pgvector 索引性能不达标 | 低 | 高 | 预设 HNSW 参数，预留调优空间 |
| 前端重构引入回归 bug | 中 | 中 | 每步提交独立 commit，保留回滚能力 |
